Arquitectura General del Sistema

El SGI se basa en una arquitectura modular, escalable y distribuida, que permite integrar diversas funcionalidades dentro de un mismo sistema. La arquitectura está dividida en tres capas principales:

  • Capa de Presentación (Frontend): Interfaz de usuario desarrollada con React, donde los usuarios interactúan con el sistema de manera intuitiva.
  • Capa de Lógica de Negocio (Backend): Desarrollo en Node.js, encargado de procesar las solicitudes y ejecutar la lógica del negocio, como la gestión de inventarios, pedidos, usuarios, entre otros.
  • Capa de Datos (Base de Datos): Base de datos SQL (MySQL o PostgreSQL), para almacenar y gestionar la información de los productos, pedidos, usuarios y transacciones.

Componentes Principales del Modelo

Los componentes clave que forman parte del modelo del SGI son:

  • Gestión de Inventarios: Módulo central que permite registrar, consultar, actualizar y eliminar productos en el inventario, y realizar el seguimiento de existencias en tiempo real.
  • Gestión de Pedidos: Módulo que permite crear, modificar y consultar pedidos de clientes y proveedores, manteniendo un registro detallado de todas las transacciones.
  • Gestión de Usuarios: Módulo de administración de roles y permisos, que permite controlar el acceso y las acciones de los usuarios dentro del sistema.
  • Reportes y Análisis: Módulo que permite generar reportes de rendimiento, ventas, inventarios y otros indicadores clave para la toma de decisiones empresariales.

Flujo de Información

El flujo de información en el SGI se gestiona de la siguiente manera:

  • Entrada de Datos: Los usuarios ingresan información a través de la interfaz, como productos, pedidos o ventas.
  • Procesamiento de Datos: La lógica de negocio procesa las solicitudes, actualiza las bases de datos y ejecuta las reglas de negocio correspondientes.
  • Salida de Datos: Los datos procesados se presentan al usuario en forma de reportes, alertas y resúmenes.

Modelo de Base de Datos

El modelo de base de datos está diseñado para garantizar la integridad de la información, optimizar el rendimiento y facilitar la escalabilidad del sistema. Este modelo incluye:

  • Entidades: Productos, Pedidos, Usuarios, Transacciones, etc.
  • Relaciones: Relación de pedidos con productos, usuarios con roles, productos con inventarios, entre otras.
