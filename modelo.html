Modelo de Desarrollo Adoptado

Para el desarrollo del Sistema de Gestión de Inventarios (SGI), se adopta una metodología ágil, específicamente Scrum. Esta metodología permite una mejor adaptación a los cambios y una entrega incremental del sistema, asegurando una evolución constante basada en la retroalimentación del cliente.

Fases del Proyecto

  • Inicio: Recolección de requerimientos, definición de alcance y recursos.
  • Planificación: Estimación de tareas, definición del backlog y planificación de sprints.
  • Desarrollo: Implementación iterativa de funcionalidades mediante sprints.
  • Pruebas: Validación del sistema, pruebas unitarias, funcionales y de integración.
  • Entrega: Presentación final del sistema, documentación y retroalimentación.

Arquitectura del Sistema

El sistema SGI se desarrollará utilizando una arquitectura cliente-servidor con las siguientes tecnologías:

  • Frontend: React.js
  • Backend: Node.js + Express
  • Base de datos: PostgreSQL o MySQL
  • APIs: RESTful para la comunicación entre frontend y backend
